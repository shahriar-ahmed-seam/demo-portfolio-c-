# Portfolio Submission Guide

This guide provides instructions on how to submit your completed portfolio website for review.

## 1. Project Overview
The portfolio is a themed engineering showcase (Pokémon-styled) built using **React, Vite, and Cloudflare Pages** (or GitHub Pages). It features:
- **HTML5 Semantic Structure**: Organized sections for Hero, Projects, Timeline, and Contact.
- **Accessibility**: Includes `alt` tags and ARIA labels.
- **CSS Styling**: Custom styling with CSS Variables and Flexbox/Grid.
- **Responsiveness**: Media queries to support mobile and desktop views.
- **Interactivity**: Dynamic data rendering and conditional components.

## 2. Preparing for Submission
Before submitting, ensure your code is pushed to a GitHub repository:

1. **GitHub Repository**: Create a public repo (e.g., `modern-portfolio`).
2. **Push Code**:
   ```bash
   git add .
   git commit -m "Initial commit"
   git push origin main
   ```
3. **Deployment**: Enable GitHub Pages in Repository Settings > Pages. Choose `GitHub Actions` or push the `dist/` folder.

## 3. Sample Submission Data
When filling out the submission form, you can use the following example:

- **Project Title**: Seam's Poke-Dex Engineering Portfolio
- **GitHub URL**: `https://github.com/YourUsername/your-repo-name`
- **Live URL**: `https://YourUsername.github.io/your-repo-name`

## 4. Grading Checklist
- [x] **HTML Structure**: Used semantic `<main>`, `<section>`, `<article>`, and `<aside>`.
- [x] **Accessibility**: Added `alt` tags to images and `aria-label` to complex sections.
- [x] **CSS Styling**: Implemented a cohesive Pokémon theme across all sections.
- [x] **JavaScript Interactivity**: Dynamic mapping of profile data, projects, and contact links.
- [x] **Responsiveness**: Verified layout on screens smaller than 768px.

Good luck with your submission!
